# Unit Test 에이전트 결과 리포트 페이지 구현 계획

## Context

[docs/reference/agent_outputs/unittest.html](../docs/reference/agent_outputs/unittest.html#L182) 의 `DATA` 상수는 Unit Test 에이전트 결과물의 합의된 인터페이스(JSON 스키마)이다. 현재 [src/app/(main)/(project)/projects/[id]/unit-test/page.tsx](../src/app/(main)/(project)/projects/[id]/unit-test/page.tsx) 는 placeholder이며, 본 작업은 그 페이지를 합의 스키마 기반으로 실제 구현한다.

대시보드 페이지가 정착시킨 상단 레이아웃(`PageToolbar` + `CommitSelector` + `Export`/`Share`)을 동일하게 사용해 에이전트 페이지간 일관성을 유지하고, 본문은 KPI / 검증 항목 / Best-of-N / Test Log 카드들로 구성한다. 테스트 파일 코드 뷰어는 사이드 패널로 분리한다 (사용자 결정).

### 사용자 결정 사항
- **코드 뷰어 배치**: 우측 패널 재사용 (Chat에서 쓰는 `RightPanel` 슬롯)
- **신택스 하이라이트**: `shiki` 추가 설치 (VS Code-grade)
- **필터 UI**: KPI Strip만 (HTML의 칩 중복 제거)

### 비결정 항목 (작업 중 자연스럽게 결정)
- 우측 패널 width 기본값 — chat과 동일한 `LayoutMode.SPLIT` (50%)
- 검색 디바운스 — 첫 컷에서는 즉시 (`useMemo` 필터링), 19건 규모라 불필요

## 작업 단계

### 1. 타입 정의 추가 — `src/types/unit-test-agent-report.ts` (신규)

`DATA` 구조를 그대로 TypeScript로 옮긴다. 기존 [src/types/unit-test-report.ts](../src/types/unit-test-report.ts) 의 `UnitTestReport`(대시보드 요약용)는 그대로 두고 **별도 타입**으로 추가해 churn 최소화.

```ts
export type UnitTestVerificationCode =
  | "PASS" | "DEFECT" | "FAIL" | "BUILD-FAIL"
  | "SKIP-NOMATCH" | "SKIP-GEN" | "SKIP-GATE" | "SKIP-TIER" | "PENDING";

export type UnitTestStatusClass =
  | "s-pass" | "s-defect" | "s-fail" | "s-nomatch"
  | "s-tier" | "s-gate" | "s-gen" | "s-covered" | "s-pending";

export type UnitTestGroupKey =
  | "PASS" | "DEFECT" | "FAILED" | "UNVERIFIED" | "NONTARGET";

export interface UnitTestAgentMeta {
  project: string; repo: string; commit: string;
  branch: string; branch_url: string; updated: string;
}
export interface UnitTestKpiGroup {
  key: UnitTestGroupKey; label: string;
  codes: UnitTestVerificationCode[];
  color: "green" | "red" | "amber" | "gray" | "blue";
  count: number;
}
export interface UnitTestVerificationItem {
  id: number; code: UnitTestVerificationCode;
  text: string; reason: string;
  slabel: string; scls: UnitTestStatusClass;
  file: string | null;
}
export interface UnitTestBestOfNRow {
  id: number; file: string; label: string; reason: string;
}
export interface UnitTestAgentReport {
  meta: UnitTestAgentMeta;
  total: number;
  groups: UnitTestKpiGroup[];
  vis: UnitTestVerificationItem[];
  files: Record<string, string>;
  bestofn: UnitTestBestOfNRow[];
  test_log: string;
  test_log_name: string;
  markdown: string;
}
```

### 5. 도메인 컴포넌트 — `src/components/unit-test/`

기존 [UnitTestListCard.tsx](../src/components/unit-test/UnitTestListCard.tsx) (대시보드용)는 **건드리지 않음**. 새 파일들:

- **`UnitTestKpiStrip.tsx`**: 6칸 그리드(전체 + 5 groups). 클릭 시 active 토글. `groups.color` 를 `StatusBadge` 의 tone에 매핑(green→emerald, red→red, amber→amber, gray→neutral, blue→blue).
  ```tsx
  interface Props {
    total: number;
    groups: UnitTestKpiGroup[];
    activeKey: UnitTestGroupKey | null;
    onChange: (key: UnitTestGroupKey | null) => void;
  }
  ```
  - 카드 스타일: [SectionCard.tsx](../src/components/shared/SectionCard.tsx#L34) 의 `bg-canvas-primary border-line-subtle rounded-lg border` 컨벤션 따름. active는 `border-line-strong` 또는 `ring-2 ring-info` 사용 (globals.css 토큰만).

- **`UnitTestVerificationCard.tsx`**: 메인 카드. `SectionCard` 사용.
  - title: `검증 항목 상세 (${vis.length}건)` + icon `ListChecks`
  - headerRight: `표시 N / total` 텍스트
  - subHeader: 검색 `Input` 한 줄
  - body: `DataTable<UnitTestVerificationItem>` ([DataTable.tsx L33](../src/components/shared/DataTable.tsx#L33))
    - 컬럼: 번호 / 검증 항목 / 분류 (StatusBadge) / 설명 / 테스트 파일 (Button variant=link)
    - `scls` → StatusBadge tone 매핑: `s-pass→emerald, s-defect→red, s-fail→red, s-nomatch→amber, s-tier→neutral, s-gate→orange, s-gen→red, s-covered→blue, s-pending→neutral`
    - 파일 셀: file null이면 `—`, 있으면 클릭 시 `openUnitTestCodeViewer({ name: file, code: report.files[file] })` 호출
  - 필터링: 부모(page)가 `activeKey` + `query` 상태 관리. `groups.codes`로 코드 매칭, `query`로 `text+reason+id` 부분일치. `useMemo`로 계산해 `DataTable.data`에 전달.

- **`BestOfNCard.tsx`**: `SectionCard` + `DataTable`. 컬럼: VI(#id) / 후보 테스트 파일 / 라벨 / 사유. 라벨 `BEST_*` → `StatusBadge tone="blue" variant="soft"`, 그 외 → `tone="neutral"`. 파일 셀 클릭 → `openUnitTestCodeViewer`.

- **`TestLogCard.tsx`**: `SectionCard` + `ScrollArea`. `<pre>` 내부에 라인별로 lightweight 분류:
  - `[FAILED]` / `❌` / `failed: [1-9]` → `text-status-red-fg`
  - `[PASSED]` / `✅` / `passed: [1-9]` → `text-status-emerald-fg`
  - `^# / ▶ / [Stage` → `text-info`
  - `*.c/*.cpp` 파일명은 `text-status-blue-fg`
  - 배경: `bg-canvas-surface-2` (다크에서도 토큰만 사용 — HTML의 `#0d1117` 같은 raw hex 금지)
  - 헤더 right에 `자동 스크롤 ON/OFF` 토글

- **`UnitTestCodeViewerPanel.tsx`**: 우측 패널 컨텐츠. 헤더(파일 탭들 + 닫기 X) + body(`CodeViewer`).
  - `usePanelStore`에서 `unitTestCodeViewer` 구독.
  - 탭은 shadcn `Tabs(variant="line")` 사용. `value=activeFile`. 각 트리거 옆에 `<X>` 아이콘 → `closeUnitTestCodeViewer(fileName)`.
  - 다중 파일 닫기 시 마지막이면 `resetRightPanel`.

### 8. Export Markdown 동작 연결

페이지 내부에 `handleExportMarkdown`:
```ts
const blob = new Blob([report.markdown], { type: "text/markdown;charset=utf-8" });
const a = document.createElement("a");
a.href = URL.createObjectURL(blob);
a.download = `unittest-${currentCommit.commit_id}.md`;
a.click();
URL.revokeObjectURL(a.href);
```
대시보드의 Export 버튼은 아직 no-op이지만, 이 페이지는 `report.markdown` 이 명시되어 있어 즉시 작동.

### 9. Share Report — 비결정

`navigator.clipboard.writeText(window.location.href)` 로 첫 컷 처리. 토스트 시스템이 없으므로 버튼 텍스트를 잠깐 "Copied" 로 토글하는 정도. (대시보드의 Share 버튼도 현재 no-op이므로 추가 작업 없이 토글만)

## Verification

1. 의존성 설치: `pnpm install` (shiki 신규)
2. `pnpm dev` 실행 → `/projects/<projectId>/unit-test` 진입
3. **상단**: CommitSelector 동작, Export Markdown 클릭 시 `.md` 파일 다운로드되고 내용이 `report.markdown` 과 일치
4. **KPI Strip**: 카드 6개 노출. "PASS" 클릭 → 검증 항목 표가 5건으로 필터링, 카드 active 표시. 다시 클릭 → 해제
5. **검색**: "팬 속도" 입력 → 매칭 행만 표시. `표시 N/19` 카운트 업데이트
6. **파일 클릭**: VI #1 행의 `test_vi_1_3_4_5_8_17_*.c` 클릭 → 우측 패널 열리고 shiki 하이라이트된 코드 표시. 라인번호 노출
7. **다중 탭**: VI #8의 같은 파일 클릭 → 새 탭 생성되지 않고 active 유지. (현재 mock은 1개 파일만 있으므로 다중 탭은 mock 데이터 확장 후 검증)
8. **탭 닫기**: `X` 클릭 → 탭 제거, 마지막이면 패널 자동 닫힘
9. **Best-of-N**: 6 행 표시, 파일 클릭 시 우측 패널 열림
10. **Test Log**: PASSED 라인은 emerald, FAILED 라인은 red, *.c 파일명은 blue로 표시
11. **다크/라이트**: 테마 토글 시 토큰이 일관 적용되는지 (특히 코드 뷰어 + 로그 카드의 배경/전경)

15. **린트**: `pnpm lint` 통과 — named export, 절대경로, raw hex 없음
