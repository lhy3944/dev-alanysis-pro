# 워크스페이스 & 어드민 메뉴 레이아웃 설계 및 컴포넌트 모듈화 구현 계획서

본 계획서는 **변경 영향 분석 결과(Code Impact Analysis)**, **커밋 히스토리(Commit History)**, 그리고 **어드민 콘솔(Admin Console) 서브메뉴 3종**의 프리미엄 레이아웃을 잡고, 향후 백엔드 API와의 실시간 데이터 연동이 100% 매끄럽게 호환되도록 하는 컴포넌트 아키텍처 및 Mock 데이터 결합 설계도입니다.

---

## 핵심 아키텍처 핵심 요약

### 1. 관심사 분리 (Separation of Concerns)
- **UI Presentational Layer**: `src/components/projects/` 및 `src/components/admin/` 폴더에 세부 컴포넌트를 고도로 모듈화하여 분리합니다. UI 컴포넌트는 오직 props로 전달받은 데이터와 UI 상태(줌, 정렬 등)만을 렌더링에 사용합니다.
- **Data Hook Layer**: `src/hooks/` 폴더 내에 `useCodeImpactData`, `useCommitHistoryData`, `useAdminMonitoringData`와 같은 전용 데이터 커스텀 훅을 생성합니다.
- **백엔드 교체 전략**: 백엔드 API가 준비되면 UI 컴포넌트의 단 한 줄의 코드 수정도 없이, 오직 이 **커스텀 훅**의 내부 fetch/React-Query 로직만 수정하여 완벽하게 데이터가 연동되도록 보장합니다.

### 2. Next.js 16 App Router 중첩 라우팅 구조
- `/admin`으로 진입 시 디폴트 페이지로 리다이렉트 처리합니다.
- 중첩 레이아웃(`admin/layout.tsx`)을 설계하여 서브 사이드바 메뉴는 공통 유지되고 하위 서브 뷰가 부드럽게 세그먼트 전환됩니다.

```
src/
├── app/
│   └── (main)/
│       ├── (project)/
│       │   └── projects/
│       │       └── [id]/
│       │           ├── code-impact/
│       │           │   └── page.tsx        <-- 영향 분석 결과 (첫 번째 이미지)
│       │           └── commit-history/
│       │               └── page.tsx        <-- 저장소 커밋 히스토리 (두 번째 이미지)
│       └── admin/
│           ├── layout.tsx                  <-- 어드민 사이드 바 레이아웃
│           ├── page.tsx                    <-- /admin/system-status로 리다이렉트
│           ├── overview/
│           │   └── page.tsx                <-- Usage Overview (미구현 상태 스켈레톤 제공)
│           ├── system-status/
│           │   └── page.tsx                <-- System Status 모니터링 (세 번째 이미지)
│           └── user-management/
│               └── page.tsx                <-- User Management (미구현 상태 스켈레톤 제공)
```

### 1. [NEW] Mock 데이터 인터페이스 및 커스텀 훅

#### 📂 [NEW] [use-code-impact-data.ts](file:///d:/Workspace/LGE/dev-alanysis-pro/src/hooks/use-code-impact-data.ts)
영향받는 모듈 토폴로지, 연관 요구사항, AI 코드 리뷰 요약 결과, 유닛 테스트 목록, 시스템 테스트 케이스 목록 등 첫 번째 화면의 전체 Mock 데이터를 담는 훅입니다.
- **데이터 명세 (Data Spec)**:
  - `topology`: 노드(id, label, type: 'source'|'affected'|'none') 및 링크(source, target) 데이터
  - `associatedRequirements`: PRM, FEATURE, SRS별 코드 매핑 리스트
  - `aiCodeReview`: 성능 최적화, 코드 가독성, 보안 취약점, 테스트 용이성 통계치 및 AI 권장사항 텍스트
  - `unitTests`: 관련 유닛 테스트 목록 (테스트명, 성공 여부, 실행 시간)
  - `systemTests`: 시스템 테스트 케이스 목록 (ID, 테스트명, 설명, 성공 여부)
  - `verificationItems`: 검증 필요 항목 (우선순위, 태그, 설명 등)

#### 📂 [NEW] [use-commit-history-data.ts](file:///d:/Workspace/LGE/dev-alanysis-pro/src/hooks/use-commit-history-data.ts)
두 번째 화면의 커밋 리스트와 상태 흐름 제어, 통계 데이터를 관리합니다.
- **인터랙티브 시뮬레이터(Simulator) 내장**:
  - `commits` 상태: 각 커밋 데이터 (해시, 저자, 날짜, 메시지, 분석 상태)
  - `stats` 상태: 주간 커밋 빈도, 분석 완료 리포트 수
  - `triggerAnalysis(commitHashes: string[])` 함수: 백엔드를 모사하여 체크된 커밋들의 분석 상태를 `Analyzing`으로 변경하고 3초 뒤에 완료(`View Result`) 처리한 뒤 완료 통계 수치를 실시간 갱신합니다.

#### 📂 [NEW] [use-admin-monitoring-data.ts](file:///d:/Workspace/LGE/dev-alanysis-pro/src/hooks/use-admin-monitoring-data.ts)
세 번째 화면의 시스템 전체 하드웨어 사용량(CPU, Active Workers, Latency, Throughput), 실시간 프로젝트 분석 큐, 실시간 시스템 에러 로그 데이터를 주기적(e.g., 5초마다 미세 난수 발생)으로 모사해 실제 Live Dashboard처럼 깜빡이고 갱신되는 고품질 훅입니다.

## 검증 계획 (Verification Plan)

### 수동 검증 항목
1. **모듈 토폴로지 인터랙션 검증**: 줌(+/-) 및 리셋 버튼 클릭 시 관계도가 부드럽게 확대/축소 및 복원되는지 확인. 노드 호버 시의 트랜지션 확인.
2. **커밋 히스토리 분석 시뮬레이터 검증**: 체크박스 선택 후 `변경점 분석` 클릭 시 `Analyzing`으로 즉각 상태가 전이되며 로딩 인디케이터가 도는지, 3초 후 완료되어 통계 카드가 올라가는지 확인.
3. **어드민 메뉴 중첩 라우트 연동 검증**: 사이드바 메뉴 클릭 시 URL이 `/admin/system-status`, `/admin/overview`, `/admin/user-management`로 완벽히 전환되며 헤더 네비게이션 엑티브 상태와 공통 서브 레이아웃이 조화롭게 결합해 유지되는지 확인.
4. **모바일/태블릿 반응형(Flex/Grid) 확인**: 뷰포트 변경 시 그리드 레이아웃의 적절한 컬럼 배치 변화 점검.
